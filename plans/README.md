## Findings considered and rejected вЂ” performance pass (do not re-audit)

Each was claimed by a subagent and **refuted by reading the cited code or the
build output**:

- **"All 7 Inter subsets (~218 KB) ship / 5 s FOIT / portal unusable"** (A/F):
  **rejected.** The built `globals.css` emits 7 `@font-face` blocks, each with a
  `unicode-range`, all `font-display: swap` (verified in
  `.output/public/assets/globals-CKuI42Oa.css`). The browser fetches only the
  subset whose range renders вЂ” an English page pulls ~48 KB (`inter-latin`), not
  218 KB, and `swap` means text is visible immediately in the fallback (no FOIT).
  The only residual is A-3 (preload the latin file). Do not file a "subset the
  fonts" plan.
- **"~150 KB of `streamdown`/`mermaid` ships to every user"** (F-1 impact):
  **rejected.** `MessageResponse` is imported nowhere, so the whole module
  tree-shakes out вЂ” `rg` for `mermaid|streamdown|LazyMotion` across every built
  chunk returns **none**. The real residual is dead code + unused deps (recorded
  above as F-1, LOW), not wire bytes.
- **"Icon maps use barrel imports that defeat tree-shaking"** (A-2): **rejected.**
  `aws-icon-map.tsx` / `azure-icon-map.tsx` use deep-path named imports
  (`aws-react-icons/icons/ArchitectureServiceвЂ¦`). Tree-shaking works; every
  bundled icon is actually rendered. The real residual is F-3вЂІ (detail-page
  over-fetch of the whole pack), kept above.
- **"`motion` is merged into the entry chunk"** (A-1 sub-claim): **rejected.**
  `motion` is its own `features-animation-*.js` (72 KB) chunk; the 446 KB entry
  is something else (analyse it in 006).
- **"Query keys churn on object identity в†’ cache misses"** (C-2/C-6): **rejected.**
  React Query hashes keys structurally (`hashKey`, stable JSON), so
  `["topics", {}]` from two call sites hit the same cache entry. Object identity
  is irrelevant.
- **"`defaultPreload: intent` + `defaultPreloadStaleTime: 0` fires redundant
  external calls on hover"** (B-1): **rejected / folded into C-1.** Preload
  re-runs the loader, but `ensureQueryData` short-circuits on any query within
  its `staleTime` (60 s / Infinity) в†’ no network. Only `contextBundle`
  (no `staleTime`) actually refetches вЂ” that is C-1, not a preload-config issue.
- **"Availability matrix needs virtualisation"** (E-1): **rejected.**
  `matrix-view.tsx:196` already applies `content-visibility:auto` +
  `contain-intrinsic-size`, and `pnpm measure:availability-render` reports
  ~6вЂ“12 ms render. Re-evaluate only if regionГ—service count grows by an order of
  magnitude.
- **"Region map recomputes an expensive SVG projection / coordinate precision is
  excessive"** (E-3): **rejected.** Path coords are integers; the projection is
  ~3 mults per marker over ~5вЂ“10 markers. Non-issue.
- **"Theme/Ask/Query context fan-out causes wide re-renders"** (E-5): **rejected.**
  Provider values are memoised and React Compiler stabilises the callbacks; no
  evidenced wide re-render.
- **"Lazy-load each service icon individually"** (F-3 fix): **rejected as
  stated** вЂ” 26 per-icon requests is worse under the proxy/TLS lens than one
  122 KB chunk. The defensible fix (detail-page icon split) is F-3вЂІ above.

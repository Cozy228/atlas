## Findings considered and rejected

- **Rate-limiter concurrency race** (auditor claim): rejected — `consume()` is
  fully synchronous (`portal/src/ask/askAtlas.ts:117-123`); Node cannot
  interleave between the read and the write. The real bugs (no reset, shared
  key) are covered by plan 003.
- **Confluence provider "doesn't handle 500s"** (auditor claim): rejected as
  stated — 401/403/404/`!ok` are all handled
  (`confluenceCloudContentProvider.ts:70-93`). The real residual gap
  (transport/JSON-parse exceptions) is plan 004.
- **`disclosure_level`/route-param validated "too late"**
  (`context-layer/src/api/httpRoute.ts:153`): not worth doing — zod validates
  before the service layer; the intermediate `Number()` is a layering nit with
  no reachable bad outcome.
- **AWS SDK "version lag"** (`@aws-sdk/* 3.1057.0`): rejected — the auditor's
  version math was wrong (3.1057 is not behind "3.600+"); no evidenced cost to
  the current pin.
- **`@streamdown/*` "possibly abandoned"**: rejected — speculative; no
  evidence gathered, package line is actively used upstream. Re-raise only
  with registry evidence.
- **Fuse index rebuilt per query change**
  (`portal/src/lib/availability-row-model.ts:76-87`): not worth doing now —
  index build over ~60 services per keystroke is negligible; revisit only if
  the service catalog grows by an order of magnitude.
- **`flatItems.indexOf(result)` O(n²)**
  (`portal/src/components/ask/ask-atlas-search.tsx:250`): real but tiny;
  fold into any future touch of that file rather than a standalone plan.
- **Proto ↔ mainline route duplication** (e.g.
  `portal/src/components/proto/catalog/adopted.tsx` mirrors mainline tabs):
  deliberate during the in-flight redesign; consolidation belongs to the
  cutover decision (see Direction below), not a pre-cutover refactor. Note:
  the three `proto/*/scale.ts` files are distinct synthetic fixtures, NOT
  copies — an auditor claim to the contrary was wrong.
- **`nitro` pinned to dated beta `3.0.260522-beta`**: monitor, don't act —
  TanStack Start 1.168.x pairs with nitro v3 betas; bumping independently
  risks SSR breakage with no current cost to staying put.
- **Exact-vs-caret pinning inconsistency across manifests**: not worth a plan;
  document a policy opportunistically when touching manifests.
- **`pnpm audit`**: could not run — the configured registry (npmmirror) has no
  audit endpoint. Dependency-vulnerability posture remains UNAUDITED; run
  `pnpm audit --prod --registry=https://registry.npmjs.org` when network
  policy allows.

Onboarding Instructions

To onboard to Meridian, follow the instructions here: the onboarding steps

App Team Console Access

Once onboarded, you can access the Meridian Console through this link: https://example.invalid/console, which will have a view like below.

App teams will get read-only access to their team's accounts as birthright permissions.

MeridianDeveloperAccess is a managed policy. It grants day-to-day developer permissions within the team's accounts.

MeridianReadOnlyAccess is a managed policy. It grants read-only visibility for auditors and support staff.

Use developer access for build teams and read-only access for reviewers.

Application Deployment to Meridian

Deployments in Meridian are done using Terraform and Harness following the platform's paved-road standard.

  • The Cloud Platform team provides reusable modules built with the platform module guidelines.

  • These modules are published in the private module registry for application teams to consume.

  • Application teams extend the modules and create their implementation code. Reference examples are in meridian-examples.

CI/CD (Harness/Terraform)
